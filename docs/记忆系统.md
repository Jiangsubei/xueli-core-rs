## 5. 存储层 (Stores)

### 5.1 MemoryStore Trait

```rust
pub trait MemoryStore {
    async fn insert(&self, item: MemoryItem) -> Result<String>;
    async fn get(&self, id: &str) -> Result<Option<MemoryItem>>;
    async fn update(&self, item: MemoryItem) -> Result<()>;
    async fn delete(&self, id: &str) -> Result<()>;
    async fn search(&self, query: &str, limit: usize) -> Result<Vec<MemoryItem>>;
    async fn get_by_user(&self, user_id: &str) -> Result<Vec<MemoryItem>>;
    async fn rebuild_index(&self) -> Result<()>;
}
```

### 5.2 SqliteConversationStore

对话历史存储，是上下文窗口的**主事实源**。

关键操作：
- `insert_conversation()` — 插入对话记录
- `get_session_history()` — 按会话获取历史
- `get_recent_messages()` — 获取最近 N 条消息
- `get_group_messages()` — 获取群聊窗口消息

### 5.3 SqliteMemoryStore

普通记忆条目存储，支持：
- 记忆 CRUD
- 衰减计算（`importance = base * (0.5 ^ (days / half_life))`）
- 冷记忆额外衰减
- 检索命中强化
- 索引重建

### 5.4 其他 Store

- **SqliteImportantMemoryStore**：重要记忆，与普通记忆同表以 `memory_type` 区分
- **SqlitePersonFactStore**：人物事实，存储用户偏好、关系等
- **SqliteFactEvidenceStore**：争议证据与软不确定信号

## 6. 检索层 (Retrieval)

### 6.1 两阶段检索流程

```
1. BM25 初排
   bm25_results = bm25_index.search(query, top_k=50)

2. 向量联想（如启用重排）
   vector_scores = vector_index.score(query, candidates)
   combined = bm25 * bm25_weight + vector * vector_weight

3. 时段匹配
   time_score = compute_time_of_day_match(current_time, memory.encoding_context)

4. 竞争抑制
   if is_suppressed(memory): score *= (1.0 - suppression_factor)

5. 归档惩罚
   if is_archived: score *= archive_penalty

6. 综合排序
   final = bm25 * w1 + importance * w2 + recency * w3 + scene * w4 + time_match * w5
```

### 6.2 BM25Index

- 中文分词：`jieba-rs`
- 索引结构：倒排索引 + 文档频率
- 支持增量更新

### 6.3 VectorIndex

- 轻量实现：char n-gram 余弦相似度
- 不依赖外部向量数据库
- 用于重排阶段语义联想

### 6.4 当前状态

- ✅ 两阶段混合评分
- ✅ 时段匹配加权

## 8. 拟人化特性

### 8.1 动态遗忘（用进废退）

- 普通记忆指数衰减：`importance = base * (0.5 ^ (days / half_life))`
- `core_fact` 3x 半衰期，`important` 1.5x，`casual` 0.7x
- 检索命中自动强化

### 8.2 脆性期保护

- 新记忆在 `consolidation_hours`（默认 48h）内不受衰减
- 离线巩固循环使用 LLM 评估后移除保护

### 8.3 提取诱发遗忘

- 召回后异步抑制同查询竞争者
- `suppression_factor=0.05`，`suppression_cooldown_retrievals=5`

### 8.4 记忆合并

- 后台按主题聚类，`merge_min_cluster_size=2`
- LLM 生成合并摘要，`merge_max_batch=5`

### 8.5 时段匹配检索

- 当前时段与记忆 `encoding_context.time_of_day` 匹配
- 加权提升同时间段记忆分数

## 9. 记忆隔离原则（强制）

- **存储路径与检索路径必须对称**：`storage_user_id` 格式为 `{platform}:{scope}:{scope_id}:{user_id}`
- **索引重建覆盖全量用户**：`rebuild_all_indices()` 必须用 `storage.get_user_ids()`
- **作用域匹配不能产生歧义**：`Group("")` 在 group_id 为空时回退到 `Private`
