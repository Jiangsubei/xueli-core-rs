## 2. BotRuntime — 运行时主协调器

`BotRuntime` 是整个系统的生命周期管理中心，对应 Python 版 `xueli/src/core/runtime.py` 的 `BotRuntime` 类。

### 2.1 职责

- 初始化和管理所有子系统组件
- 消息处理主循环（单次触发、run-to-completion）
- 回复发送协调（通过 `ReplySender`）
- 状态机管理（Created → Initializing → Running → Stopping → Stopped）
- 背压机制（消息缓冲、中断协调、精力感知冷却）
- 主动分享调度（`ProactiveShareScheduler`）

### 2.2 消息处理主循环

```
1. Runtime 前置：过滤 bot 自消息
2. GroupMessageCollector：群消息预落库（仅非 bot）
3. MessageHandler.handle()
   - @mention / 私聊：直接 reply
   - 群聊非 @：消息缓冲 → TimingGate 触发 → reply / wait / ignore
4. REPLY 时：build_message_context → ReplyAgent (tool-calling)
5. ReplySender 发送 → record_reply_sent → MemoryFlowService
```

### 2.3 背压机制

| 层级 | 机制 | 说明 |
|------|------|------|
| Session | 会话串行 | per-session worker，同会话顺序处理 |
| Runtime | 消息缓冲 | 动态阈值（频率 + 心情 + 饱和度），攒 N 条非 @ 消息才触发 TimingGate |
| Runtime | 中断协调 | `max_interrupt_count=2`（@/私聊跳过），新消息可打断推理 |
| Runtime | 状态机 | RUNNING / WAITING / STOPPED，STOPPED 含精力感知冷却 |
| Runtime | group_wait_window | 可配秒数，wait 窗口超时后重新评估 |

### 2.4 关键数据结构

```rust
pub struct BotRuntime {
    pub config: Arc<XueliConfig>,
    pub metrics: Arc<RwLock<RuntimeMetrics>>,
    state: Arc<RwLock<RuntimeState>>,
}

pub enum RuntimeState {
    Created,
    Initializing,
    Running,
    Stopping,
    Stopped,
}
```

## 3. XueliConfig — 配置系统

对应 Python 版 `xueli/src/core/config.py`。

### 3.1 配置结构

```rust
pub struct XueliConfig {
    pub model: ModelConfig,           // AI 模型配置
    pub reply: ReplyConfig,           // 回复行为配置
    pub memory: MemoryConfig,         // 记忆系统配置
    pub timing_gate: TimingGateConfig, // TimingGate 配置
    pub session: SessionConfig,       // 会话配置
    pub emoji: EmojiConfig,           // 表情系统配置
    pub proactive_share: ProactiveShareConfig, // 主动分享配置
}
```

### 3.2 关键配置项

**ModelConfig:**
- `primary_model` / `light_model` / `vision_model` — 模型名
- `api_base` / `api_key` — API 端点
- `temperature` / `max_tokens` — 生成参数
- `context_window` — 上下文窗口大小（默认 128000）

**ReplyConfig:**
- `context_token_budget_ratio` — 输入预算比例（默认 0.7）
- `max_message_length` — 单条消息最大长度
- `segmented_reply_enabled` / `max_segments` — 分段发送
- `followup_delay` — 分段间隔

**MemoryConfig:**
- `db_path` — SQLite 数据库路径
- 衰减参数（half_life、forget_threshold、cold_threshold）
- 巩固参数（consolidation_hours）
- 合并参数（merge_min_cluster_size）

**TimingGateConfig:**
- `group_reply_decision` — 群聊决策模型配置（未配置时退回规则路径）
- `group_wait_window_seconds` — wait 等待窗口

## 4. EventDispatcher — 事件分发器

对应 Python 版 `xueli/src/core/dispatcher.py`。

### 4.1 职责

- 注册预处理器、消息处理器、通知处理器
- 将原始事件分发到对应处理器链
- 预处理 → 分发 → 后处理

### 4.2 事件流向

```
原始数据 → dispatch()
  → 预处理器链
  → 根据 post_type 分发（message → MessageHandler, notice → NoticeHandler）
  → 后处理器链
```

## 7. MoodEngine — 情绪引擎

对应 Python 版 `xueli/src/core/mood_engine.py`。

### 7.1 职责

- 维护 `valence`（愉悦度）和 `energy`（精力）两个维度
- 滑动平均更新情绪状态
- 情绪影响 TimingGate 触发阈值和冷却时长

### 7.2 关键逻辑

- **精力影响阈值**：`energy` 低时抬门槛（精力 0.2 → +3.8 条），`valence` 消极时加惩罚
- **STOPPED 冷却**：`30.0 + 200.0 * exp(-3.0 * energy)` 秒
- **群聊饱和度**：对数增长，无硬上限

### 7.3 当前状态

Rust 版已实现基本 `MoodEngine` 结构，待集成到 TimingGate 触发阈值计算。

## 8. ChatScope — 会话模态

```rust
pub enum ChatScope {
    Private,           // 一对一对话
    Group(String),     // 多用户会话（群聊，含群 ID）
}
```

**强制规则：**
- 所有表示会话模态的逻辑必须使用 `ChatScope` 枚举
- 禁止硬编码字符串比较
- `Group("")` 在 group_id 为空时需回退为 `Private`

## 9. 错误类型

```rust
pub enum XueliError {
    Config(String),
    AI(String),
    Memory(String),
    Platform(String),
    Internal(String),
}
```

基于 `thiserror`，核心逻辑路径必须返回 `Result<_, XueliError>`。`unwrap()` / `expect()` 仅允许在初始化阶段和测试中使用。
