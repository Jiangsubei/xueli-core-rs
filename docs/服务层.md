## 2. DefaultAIClient — AI 客户端

基于 HTTP + OpenAI 兼容 API 的 `AIClient` trait 默认实现。

### 2.1 当前实现

```rust
pub struct DefaultAIClient {
    config: Arc<ModelConfig>,
    client: reqwest::Client,
}

impl AIClient for DefaultAIClient {
    async fn chat_completion(&self, request: &ChatCompletionRequest)
        -> Result<ChatCompletionResponse, String>;
}
```

### 2.2 功能

- 构建 OpenAI 兼容请求（model、messages、temperature、max_tokens）
- 发送 HTTP POST 到 `{api_base}/chat/completions`
- 解析响应 JSON（choices[0].message.content + usage）
- 120 秒超时

## 3. VisionClient — 视觉客户端

调用 VLM（Vision Language Model）进行图片理解。对应 Python 版 `xueli/src/services/vision_client.py`。

### 3.1 职责

- 图片理解（单图/多图）
- 表情情绪分类
- 图片描述缓存

### 3.2 图片描述口径

- **单张图片**：使用该图的逐图描述 `[图片] {per_image_description}`
- **多张图片**：优先使用合并描述 `[图片] {merged_description}`；若合并描述缺失，将逐图描述拼接
- **识别失败**：所有描述均缺失时回退为 `[图片]未成功识别`

### 3.3 当前状态

- ✅ VLM API 调用（支持 `image_url` 和 `base64` 两种图片格式）
- ✅ 多图合并描述
- ✅ 表情情绪分类（加载 `vision_emotion.prompt`）
- [ ] 图片描述缓存（`SignalCache` 集成）

## 5. TokenCounter — Token 计数器

基于 tiktoken-rs 的统一 token 计数与预算管理。对应 Python 版 `xueli/src/services/token_counter.py`。

### 5.1 当前实现

```rust
pub struct TokenCounter {
    bpe: tiktoken_rs::CoreBPE,
}

impl TokenCounter {
    pub fn new_cl100k() -> Result<Self>;    // GPT-4 编码
    pub fn new_o200k() -> Result<Self>;     // GPT-4o 编码
    pub fn count(&self, text: &str) -> usize;
    pub fn count_messages(&self, messages: &[ChatMessage]) -> usize;
}
```

### 5.2 使用场景

集成在所有 LLM 调用点：

| 调用点 | 用途 |
|--------|------|
| TimingGate | 截断历史消息 |
| Planner | 截断规划上下文 |
| ReplyAgent | 截断系统 prompt + 对话历史 |
| SignalOrchestrator | 截断信号计算上下文 |
| EmojiReplyService | 截断表情追评上下文 |
| TwoStageRetriever | 截断重排候选 |

### 5.3 配置项

- `context_token_budget_ratio` — 输入预算比例（默认 0.7）
- `token_encoding` — tiktoken 编码名（默认 `cl100k_base`）
- 各模型配置节 `context_window` — 模型上下文窗口（默认 128000）

- [ ] `trim_to_budget(messages, budget_ratio)` — 按预算比例截断

## 6. ModelInvocationRouter — 模型调用路由器

根据调用类型路由到不同模型配置。对应 Python 版 `xueli/src/core/model_invocation_router.py`。

### 6.1 调用类型

| 类型 | 说明 | 默认模型 |
|------|------|---------|
| `TimingGate` | 时机判断 | `light_model` |
| `Planner` | 回复规划 | `light_model` |
| `ReplyAgent` | 回复生成 | `primary_model` |
| `SignalOrchestrator` | 信号计算 | `light_model` |
| `VisionAnalysis` | 图片理解 | `vision_model` |
| `MemoryExtraction` | 记忆提取 | `light_model` |
| `EmojiClassification` | 表情分类 | `vision_model` |

### 6.2 职责

- 统一管理多模型 API 端点
- 记录每次调用的 token 用量
- 模型故障时提供 fallback 路径

### 6.3 当前状态
