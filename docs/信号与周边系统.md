### 2. SignalOrchestrator — 统一信号解析器

统一信号解析器：L1 缓存 → L2 存储 → 模型 → 降级回退。对应 Python 版 `xueli/src/handlers/signals/orchestrator.py`。

#### 2.1 信号类型

| 信号类型 | 说明 | 输入源 |
|---------|------|--------|
| 用户情绪 | 当前对话情绪标签 | TimingGate 输出 |
| 反馈偏好 | 用户对回复风格的反馈 | feedback_triage 输出 |
| 叙事信号 | 长期相处脉络 | narrative_self 后台更新 |
| 元认知 | 自我监控信号 | 结构化信号聚合 |
| 视觉信号 | 图片理解结果 | VisionClient 输出 |
| 角色自适应 | 角色卡调整建议 | character_adaptation 输出 |

#### 2.2 信号获取流程

```
1. L1 内存缓存（short TTL ~60s）
2. L2 存储（SignalStore SQLite）
3. LLM 计算（如有 AI 模型配置）
4. 降级回退（缺信号时默认为不注入）
```

#### 2.3 关键原则

- **语义信号失败原则**：LLM 失败时不使用关键词/规则伪造语义信号，直接不注入该部分提示词
- **用户情绪只接受 LLM 或已有结构化信号**：不使用关键词估计

#### 2.1 角色卡内容

| 字段 | 说明 |
|------|------|
| `stable_signals` | 稳定信号（长期偏好） |
| `explicit_feedback` | 显式反馈（最近 50 条） |
| `relationship_profile` | 关系画像（trust_level、formality_distance、emotional_safety） |
| `emotional_history` | 情绪历史 |

#### 2.2 角色自适应

- 当信号变化 ≥ 3 条或间隔 ≥ 300s 时触发

- 结果缓存到 `persona_hints_cache`

### 2. EmojiManager — 表情管理器

协调贴纸持久化和空闲时情绪分类。对应 Python 版 `xueli/src/emoji/manager.py`。

#### 2.1 采集规则

- 普通图片（image sub_type=0）：只做视觉理解，不入表情仓库
- 表情贴纸（image sub_type≠0,4,9）：自动采集入库
- SHA256 去重
- VLM 标注情绪标签

#### 2.2 空闲分类

- 消息空闲 `idle_seconds_before_classify`（默认 45s）后触发
- 按 `classification_interval_seconds`（默认 30s）间隔执行
- 仅在工作时间窗口内执行（`classification_windows` 配置）

### 2. ProactiveShareScheduler — 主动分享调度器

后台记忆消化每产生一条 insight 时，自动写入一条主动分享，由调度器定时发送。对应 Python 版 `xueli/src/core/proactive_share_scheduler.py`。

#### 2.1 配置

- `enabled`：默认 `false`（关闭）
- `idle_hours`：用户空闲 N 小时后才发送（默认 24h）
- `cooldown_hours`：两次发送最小间隔（默认 6h）
- `max_per_day`：每日最大发送数（默认 3）
- `time_range`：允许发送的时间窗口（默认 09:00-22:00）
- `check_interval_seconds`：检查间隔（默认 600s）

#### 2.2 发送路径

- `BotRuntime.send_proactive_share()` 创建 `ReplyAction` 经由 `PlatformAdapter` 发出
- 发送失败不会 `mark_sent`，下次重试
